¿Cómo funciona este proyecto?

1. El Problema

La identificación automática de lenguajes de programación es un desafío clásico de Natural Language Processing (NLP). Cada lenguaje tiene su propia gramática y "huella digital" sintáctica.

2. Entradas y Preprocesamiento

El input es un bloque de texto que se somete a un proceso de Tokenización. Esto convierte cada palabra o símbolo en un índice numérico para que la red pueda procesarlo.

  • Secuencias: Convertimos el texto en una lista de números.
  • Padding: Todas las entradas se nivelan a un tamaño fijo de 500 tokens.

3. Lo que sucede "en el medio"

Los datos pasan por una capa de Embedding que agrupa palabras similares en un espacio de 32 dimensiones. Luego, la capa Bi-LSTM analiza la secuencia en ambas direcciones para entender el contexto.

4. Salidas

La capa final genera 5 valores numéricos. Aplicamos Softmax para que estos valores se conviertan en probabilidades que suman 100%, indicando qué tan seguro está el modelo de pertenecer a C, C#, Java, Javascript o Python.

Arquitectura RNN Bi-LSTM

Para procesar código, que es una secuencia de lógica pura, implementamos una Red Neuronal Recurrente (RNN) con celdas LSTM. Esta estructura actúa como una "Capa de Memoria" que, gracias a su diseño Bidireccional, permite a las neuronas leer el fragmento en ambos sentidos y capturar el contexto sintáctico completo de cada token.

Cómo se aplica: Esta lectura dual permite al modelo identificar patrones complejos, como la apertura de llaves {} en Java o la indentación en Python, reconociendo la estructura del lenguaje más allá de simples palabras aisladas.

Word Embedding

Dado que una IA no puede leer texto de forma nativa, esta capa actúa como un puente fundamental. El Word Embedding convierte cada palabra en un vector numérico denso dentro de un espacio de 32 dimensiones, donde la cercanía física entre vectores representa una similitud de significado real.

Cómo se aplica: Mediante este mapeo, la red descubre por sí sola que términos como function (JS) y def (Python) ocupan lugares similares en el espacio vectorial porque cumplen el mismo rol lógico dentro de sus respectivos ecosistemas.

Optimización Adam

Para que la red aprenda correctamente de sus errores, utilizamos Adam, un algoritmo de optimización adaptativo que ajusta la fuerza de las conexiones neuronales según los errores cometidos, acelerando el aprendizaje y logrando resultados precisos en tiempo récord.

Cómo se aplica: Durante el entrenamiento detallado en training.py, este algoritmo minimiza la "sorpresa" del modelo, refinando su capacidad de distinguir entre lenguajes muy similares como C y C# basándose en miles de ejemplos reales procesados.

Padding y Softmax

El Padding normaliza todas las secuencias de código a una longitud fija de 500 tokens para que la red reciba datos uniformes. A continuación, la función Softmax traduce los valores abstractos de las neuronas de salida en probabilidades entre 0% y 100% que suman exactamente 100%.

Cómo se aplica: Este paso final es el que nos permite presentarte un ranking de confianza, donde el Softmax nos indica qué tan seguro está el modelo de que el código ingresado pertenece a uno de los 5 lenguajes soportados.
